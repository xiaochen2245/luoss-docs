# 常见问题排查

## 环境问题

### hccllib.so 文件无法找到

如果在启动训练任务时遇到 `hccllib.so` 文件无法找到的错误，使用以下命令修复：

```bash
source /usr/local/Ascend/ascend-toolkit/set_env.sh
```

### 环境变量初始化问题

::: warning 警告
对于单节点多卡训练，**不要使用** `source /models/share/init_env.sh` 命令进行环境变量的初始化，这仅适用于多节点分布式训练场景。
:::

## 任务提交问题

### 队列不可用

使用 `ktp queues` 命令查看自己账号可以使用的队列，然后对应调整任务配置。

### 任务提交失败

1. 检查 YAML 配置文件中的 `queue` 参数是否正确
2. 确认账号是否有该队列的使用权限
3. 查看是否有足够的资源配额

## 训练问题

### 分布式训练通信失败

1. 确认所有节点的网络互通
2. 检查 `MASTER_IP` 和 `MASTER_PORT` 是否配置正确
3. 验证 `TOTAL_NODES` 和 `CURRENT_NODE_RANK` 是否设置正确

### 镜像拉取失败

确认镜像路径是否正确，以及镜像仓库的网络连接是否正常。

## 推理问题

### 推理服务启动缓慢

模型第一次推理时会进行参数激活，属于正常现象，请耐心等待。

### 请求超时

1. 检查 master 和 worker 节点之间的网络连接
2. 确认使用端口情况，如 `8010` 是否被正确开放

## 获取帮助

如果遇到其他问题，请：

1. 查看任务日志：`ktp logs <job_id>`
2. 参考 [ktp CLI 文档](../guide/cli)
3. 联系平台管理员
