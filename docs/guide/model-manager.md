# 数据存储

数据存储页面展示您的用户存储使用情况以及开发环境的 overlay 持久化存储。

## 功能概述

| 功能 | 说明 |
|------|------|
| **存储概览** | 查看存储配额和使用情况 |
| **文件管理** | 浏览和管理存储文件 |
| **数据上传** | 上传数据到存储 |

## 存储概览

### 查看配额

| 信息 | 说明 |
|------|------|
| 已用空间 | 当前使用的存储空间 |
| 总配额 | 分配的存储配额 |
| 使用率 | 已用/总量百分比 |

### 存储类型

平台使用两种存储机制：

| 存储类型 | 用途 | 路径 | 说明 |
|----------|------|------|------|
| **共享用户存储** | 代码、数据集、模型 | `/config/workspace` | PVC 挂载，所有环境共享 |
| **Overlay 持久化存储** | 容器全量持久化 | 容器内 `/`（根文件系统） | 256G 稀疏镜像，按需分配 |

### 存储路径

用户存储挂载到开发环境和训练任务：

| 环境 | 挂载路径 | 说明 |
|------|----------|------|
| 开发环境 - workspace | `/config/workspace` | 共享 PVC，跨环境共享 |
| 开发环境 - overlay | 容器根文件系统 | 256G 全量持久化（pip/apt install 等均保留） |
| 训练任务 | `/models` | 训练数据与输出 |

::: tip 共享存储
所有开发环境共享同一个用户存储 PVC，数据在不同环境间互通。适合存放代码、数据集和模型文件。
:::

::: info Overlay 存储
开发环境使用 Loop Device + OverlayFS 实现全量持久化。容器内的所有修改（安装包包、系统配置等）存储在宿主机 `/mnt/model/overlay/{用户名}/{环境名}/` 目录下。环境重启后所有修改保留，删除环境时自动清理。
:::

## 目录结构

推荐的目录结构：

```
/config/workspace/     # 开发环境共享存储（推荐存放代码和数据）
├── datasets/          # 数据集
│   ├── train/         # 训练数据
│   └── val/           # 验证数据
├── models/            # 预训练模型
│   └── llama-3-8b/
├── output/            # 训练输出
│   └── exp-001/
└── projects/          # 项目代码

/ (overlay 根文件系统)    # 全量持久化，256G
├── usr/local/lib/     # pip 安装的包（持久化）
├── usr/bin/           # 系统工具（持久化）
└── etc/               # 系统配置（持久化）
```

## 使用建议

1. **代码和数据放 workspace**：`/config/workspace` 路径下的内容跨环境共享，适合存放项目代码和数据集
2. **工具安装随意装**：overlay 持久化确保 pip/apt 安装的包包在环境重启后不丢失
3. **定期清理**：删除不需要的文件，overlay 存储总容量 256G
4. **备份重要数据**：定期下载备份关键数据

## 常见问题

### Q: 存储空间不足？

A: 清理不需要的文件。overlay 存储可在环境内通过 `du -sh /*` 查看各目录占用。共享 PVC 存储可联系管理员增加配额。

### Q: 如何在不同环境间共享文件？

A: 所有环境共享同一 `/config/workspace` 存储，直接访问对应路径即可。

### Q: pip install 的包在环境重启后还在吗？

A: 在的。开发环境采用全量持久化，容器根文件系统的所有修改（包括 pip/apt 安装）都会保留。

### Q: overlay 存储实际占用多少空间？

A: overlay 使用稀疏镜像文件（storage.img），虚拟大小 256G，实际占用按使用量增长。可通过宿主机 `du -sh /mnt/model/overlay/{用户名}/{环境名}/` 查看实际占用。
