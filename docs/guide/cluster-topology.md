# 集群拓扑

集群拓扑页面展示集群 NPU 资源的实时状态和可用性，帮助用户了解资源分布，利用碎片化 NPU 资源。

<FeatureBadge status="stable" />

## 功能概述

| 功能 | 说明 |
|------|------|
| **NPU 拓扑视图** | 可视化展示各节点 NPU 分布 |
| **实时状态** | 显示 NPU 占用和可用状态 |
| **显存利用率** | 查看各 NPU 的 HBM 使用情况 |
| **资源统计** | 汇总总 NPU、可用 NPU、已占用数量 |

## 访问集群拓扑

1. 点击左侧菜单 **集群拓扑**
2. 查看集群资源概览和节点状态矩阵

## 资源概览

页面顶部展示集群整体资源统计：

| 指标 | 说明 |
|------|------|
| 总节点数 | 集群中计算节点数量 |
| 可用 NPU / 总 NPU | 当前可用的 NPU 数量和总量 |
| 已占用 | 当前正在使用的 NPU 数量 |
| 平均显存利用率 | 所有 NPU 的平均 HBM 使用率 |

## 节点状态矩阵

### 节点卡片

每个节点以卡片形式展示，包含：

- **节点名称**：计算节点标识
- **NPU 状态行**：以可视化方式展示该节点所有 NPU

### NPU 状态

每个 NPU 以色块形式展示：

| 颜色 | 状态 | 说明 |
|------|------|------|
| 🟢 绿色 | 空闲 | NPU 可用，未被占用 |
| 🔴 红色 | 占用 | NPU 正在被任务使用 |
| ⚪ 灰色 | 未知 | 无法获取 NPU 状态 |

### 显存利用率

色块中的数字表示该 NPU 的 HBM（高带宽内存）利用率百分比：

- **0-30%**：低利用率
- **30-70%**：中等利用率
- **70-100%**：高利用率

## 利用碎片化资源

### 识别碎片资源

通过集群拓扑可以快速识别：

1. **空闲节点**：所有 NPU 都是绿色的节点
2. **部分占用节点**：存在绿色（空闲）NPU 的节点
3. **碎片分布**：不同节点上分散的可用 NPU

### 碎片资源利用策略

#### 1. 单卡任务

如果只需要 1 个 NPU，可以选择任意空闲 NPU 所在节点：

```bash
# 在训练任务中选择单 Task 模式
# NPU 数量设置为 1
```

#### 2. 多卡单节点任务

需要多个 NPU 在同一节点时：

1. 查找有足够连续绿色 NPU 的节点
2. 确保所需 NPU 数量 ≤ 单节点空闲 NPU 数

#### 3. 多节点分布式任务

需要跨节点使用 NPU 时：

1. 统计各节点空闲 NPU 数量
2. 选择空闲 NPU 较多的节点组合
3. 在训练任务中使用自定义模式配置多 Task

::: tip 最佳实践
提交任务前先查看集群拓扑，选择资源充足的节点，可以减少任务排队等待时间。
:::

## 刷新数据

- **手动刷新**：点击右上角 **刷新数据** 按钮
- **自动刷新**：页面每 30 秒自动刷新数据

## 使用场景

### 场景 1：快速找到可用资源

当需要立即启动任务时：

1. 打开集群拓扑
2. 查看哪些节点有绿色（空闲）NPU
3. 根据空闲 NPU 数量调整任务配置

### 场景 2：规划大规模训练

启动大规模分布式训练前：

1. 查看当前可用 NPU 总数
2. 确认各节点空闲情况
3. 决定是否等待更多资源释放

### 场景 3：监控资源使用

定期查看：

- 集群整体利用率
- 哪些节点负载较高
- 是否有资源浪费

## 注意事项

::: warning 数据延迟
拓扑数据每 30 秒刷新一次，显示的状态可能有短暂延迟。实际调度时，资源状态可能已发生变化。
:::

::: info 资源占用说明
显示"占用"的 NPU 可能正在运行：
- 其他用户的训练任务
- 您自己的开发环境
- 系统服务或调试任务
:::
