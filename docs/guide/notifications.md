# 站内信通知

LuoSS 平台提供了完善的站内信通知系统，帮助您实时掌握资源状态、任务执行情况和系统告警等重要信息。

## 概述

站内信通知系统基于 WebSocket 实现实时推送，当您在线时会立即收到新通知；离线时的消息会被暂存，在您下次连接时送达。

## 访问通知中心

在页面右上角导航栏中，点击铃铛图标 🔔 即可打开通知中心。

- 铃铛图标上的红色徽章显示未读通知数量
- 有新通知时，铃铛图标会有脉冲动画提示

## 通知类型

平台支持以下类型的通知：

| 类型 | 说明 | 严重级别 |
|------|------|----------|
| Pod 错误 | 容器 Pod 运行异常 | Error |
| 环境错误 | 开发环境启动或运行失败 | Error |
| 训练任务错误 | 训练任务执行异常 | Error |
| 配额警告 | 资源配额即将耗尽或超限 | Warning |
| 系统告警 | 平台系统级别的重要通知 | Warning/Error |
| 抢占通知 | 任务被高优先级任务抢占 | Warning |
| 监控告警 | Prometheus 告警规则触发（仅管理员可见） | Info/Warning/Error |

## 严重级别

每条通知都有对应的严重级别，以不同颜色标识：

- **Error（错误）** <span style="color: #ff4d4f">红色</span> - 需要立即关注和处理的问题
- **Warning（警告）** <span style="color: #faad14">橙色</span> - 需要注意的潜在问题
- **Info（信息）** <span style="color: #1890ff">蓝色</span> - 一般性信息通知

## 通知中心功能

### 查看通知列表

通知中心提供两种视图：

- **全部** - 显示所有通知
- **未读** - 仅显示未读通知

通知列表按时间倒序排列，最新的通知显示在最上方。

### 通知详情

每条通知包含以下信息：

- **标题** - 通知的简要描述
- **消息内容** - 详细的说明信息
- **严重级别图标** - 标识通知的重要程度
- **时间** - 事件发生时间
- **关联资源** - 如命名空间、Pod 名称、任务名称等（如适用）

### 标记已读

您可以通过以下方式管理通知的已读状态：

1. **单条标记已读** - 点击通知条目右侧的"标记已读"按钮
2. **全部标记已读** - 点击通知中心顶部的"全部已读"按钮

### 删除通知

点击通知条目右侧的删除按钮即可删除该通知。

::: tip 提示
删除通知仅从您的通知列表中移除，不影响实际的任务或资源状态。
:::

## 实时推送机制

### WebSocket 连接

通知系统使用 WebSocket 实现实时推送：

- 登录后自动建立连接
- 支持断线重连
- 心跳检测保持连接活跃

### 离线消息

当您不在线时，系统会暂存您的通知消息（最多 100 条），在您下次登录并建立连接后自动推送。

### 消息去重

系统会对相似的通知进行去重处理，在 30 分钟内相同来源的重复通知不会重复推送，避免通知轰炸。

## 通知自动清理

为了保持系统整洁，已读通知会在保留一定时间后自动清理。

## 最佳实践

1. **及时处理错误通知** - 错误级别的通知通常表示任务失败或资源异常，建议尽快查看和处理

2. **关注配额警告** - 配额警告提示您的资源使用接近限制，请及时调整或申请扩容

3. **定期清理通知** - 养成定期查看和处理通知的习惯，避免通知堆积

4. **配合其他功能** - 通知中通常会包含相关资源的名称，可以结合"我的作业"、"开发环境"等页面快速定位问题

## 常见问题

### 为什么没有收到实时通知？

请检查以下几点：

1. 确认您已登录系统
2. 检查浏览器是否允许 WebSocket 连接
3. 检查网络连接是否正常
4. 刷新页面重新建立连接

### 通知数量与实际不符？

通知数量基于数据库中的未读记录计算。如果您在其他设备上已读，当前设备会同步更新。刷新页面可以获取最新状态。

### 某些通知没有收到？

可能的原因：

1. **消息去重** - 短时间内相似的通知会被合并
2. **权限限制** - 您只能收到与您相关资源的通知
3. **离线时间过长** - 离线消息队列有容量限制，超出的消息会被丢弃

## 管理员广播通知

管理员可以向所有用户或指定用户发送广播通知。

### 发送广播

1. 进入管理后台的通知管理功能
2. 点击 **发送广播**
3. 填写通知标题和内容
4. 选择接收范围（全部用户或指定用户）
5. 确认发送

### 通过 API 发送

```bash
POST /api/v1/admin/notifications/broadcast
Authorization: Bearer <admin-token>
Content-Type: application/json

{
  "title": "系统维护通知",
  "message": "平台将于今晚 22:00 进行维护升级",
  "severity": "warning"
}
```

广播通知会推送到所有目标用户的站内信中。
