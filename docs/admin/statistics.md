# 统计分析

统计分析页面展示平台的整体使用统计和分析报表，支持计费功能。

<FeatureBadge status="stable" />

## 功能概述

| 功能 | 说明 |
|------|------|
| **使用趋势** | 资源使用趋势图表 |
| **用户排行** | 用户资源使用排名 |
| **使用时长统计** | 用户使用时长分析 |
| **账单统计** | NPU 计费和账单管理 |
| **报表导出** | 导出统计报表 |

## 使用趋势

### 时间范围选择

| 快捷选项 | 说明 |
|----------|------|
| 今日 | 今天 00:00 至现在 |
| 本周 | 本周一至现在 |
| 本月 | 本月 1 日至现在 |
| 最近 7 天 | 过去 7 天（默认） |
| 最近 30 天 | 过去 30 天 |
| 自定义 | 自定义时间范围 |

### 指标选择

| 指标 | 说明 |
|------|------|
| CPU 使用量 | 集群 CPU 使用量（核） |
| 内存使用量 | 集群内存使用量（GB） |
| NPU 使用量 | 集群 NPU 使用量（个） |

### 趋势图表

图表展示选定时间范围和指标的趋势曲线：

- X 轴：时间
- Y 轴：资源使用量
- 支持缩放和拖拽查看详情

## 用户排行

| 列 | 说明 |
|----|------|
| 排名 | 使用量排名 |
| 用户 | 用户名 |

| 内存使用 | 总内存使用量（GB） |
| NPU 使用 | 总 NPU 使用量（个） |

### 排序功能

- 支持按 CPU、内存、NPU 使用量排序
- 点击表头切换排序

## 使用时长统计

### 统计说明

使用时长 = 资源量 × 时间

例如：
- 使用 2 核 CPU 运行 3 小时 = 6 核·时
- 使用 4 个 NPU 运行 2 小时 = 8 卡·时

### 数据表格

| 列 | 说明 |
|----|------|
| 排名 | 使用时长排名 |
| 用户 | 用户名 |
| CPU 使用时长 | 核·时 |
| 内存使用时长 | GB·时 |
| NPU 使用时长 | 卡·时 |
| 采样点数 | 监控数据点数 |

### 导出时长统计

点击 **导出 CSV** 按钮导出使用时长数据。

## 账单统计

账单统计标签页提供 NPU 计费功能，按月生成用户账单。

### 设置单价

1. 点击 **设置单价** 按钮
2. 输入 NPU 单价（元/卡时）
3. 点击 **确认设置**

::: info 单价说明
- 单价单位：元/卡时
- 修改单价后，新生成的账单使用新单价
- 已生成的账单不受影响
:::

### 生成账单

1. 选择 **年份** 和 **月份**
2. 点击 **生成账单** 按钮
3. 系统自动计算每个用户的 NPU 使用时长并生成账单

::: warning 注意
- 每月只能生成一次账单
- 重复生成会覆盖已有账单
- 建议在每月初生成上月账单
:::

### 账单汇总

| 指标 | 说明 |
|------|------|
| 总账单数 | 生成的账单数量 |
| 总 NPU 卡时 | 所有用户的 NPU 使用时长总和 |
| 总费用 | 所有账单的费用总和 |
| 待支付 | 状态为"待支付"的账单费用 |

### 账单列表

| 列 | 说明 |
|----|------|
| 用户 | 用户名 |
| 账单周期 | 年-月 |
| NPU 卡时 | 使用时长 |
| 费用 | 费用（元） |
| 状态 | 待支付/已支付/已豁免 |
| 生成时间 | 账单生成时间 |

### 账单状态

| 状态 | 说明 | 颜色 |
|------|------|------|
| 待支付 | 等待支付 | 橙色 |
| 已支付 | 已完成支付 | 绿色 |
| 已豁免 | 费用豁免 | 灰色 |

### 筛选和导出

- **状态筛选**：点击状态列的筛选按钮，选择要查看的状态
- **导出 CSV**：点击 **导出 CSV** 按钮导出账单数据

导出文件包含：
```csv
用户名,年,月,NPU卡时,费用(元),状态,生成时间
```

## 报表导出

### 导出步骤

1. 点击右上角 **导出数据** 按钮
2. 选择时间范围
3. 选择数据粒度：

| 粒度 | 说明 |
|------|------|
| 原始数据 | 不聚合 |
| 按小时聚合 | 每小时汇总 |
| 按天聚合 | 每天汇总 |
| 按周聚合 | 每周汇总 |
| 按月聚合 | 每月汇总 |

4. 点击 **开始导出**
5. 等待生成完成后自动下载

### 导出格式

- 格式：CSV
- 编码：UTF-8（带 BOM）
- 包含所有统计指标

## 数据刷新

- **手动刷新**：点击 **刷新数据** 按钮
- **缓存机制**：页面数据缓存 5 分钟

## 最佳实践

### 1. 定期检查资源使用

- 每周查看用户排行
- 关注高消耗用户
- 及时调整配额

### 2. 账单管理流程

1. 月初设置 NPU 单价
2. 生成上月账单
3. 审核账单数据
4. 通知用户缴费
5. 标记已支付/已豁免

### 3. 成本优化

- 分析使用趋势，识别低效使用
- 对长期闲置用户减少配额
- 利用时长统计进行成本分摊

## 常见问题

### Q: 账单数据可以修改吗？

A: 已生成的账单不能直接修改，但可以重新生成覆盖。

### Q: 使用时长统计和账单 NPU 卡时有什么区别？

A: 使用时长统计是原始监控数据，账单 NPU 卡时是按月汇总后的计费数据。

### Q: 如何处理账单异常？

A: 检查监控数据是否正常，如有问题可重新生成账单。
