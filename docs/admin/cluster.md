# 集群监控

集群监控页面展示 Kubernetes 集群的整体状态和资源使用情况，帮助管理员实时掌握集群健康状况。

<FeatureBadge status="stable" />

## 功能概述

| 功能 | 说明 |
|------|------|
| **集群概览** | 节点状态、NPU 占用、资源利用率 |
| **利用率报告** | 时间范围内的资源利用率统计和分析 |
| **节点详情** | 单个节点的详细信息和运行 Pod |

## 集群概览

### 资源统计卡片

页面顶部展示集群整体资源统计：

| 指标 | 说明 |
|------|------|
| 总节点数 | 集群中计算节点数量 |
| 可用 NPU / 总 NPU | 当前可用和总量 |
| 已占用 | 正在使用的 NPU 数量 |
| 平均显存利用率 | 所有 NPU 的平均 HBM 使用率 |

### 节点状态矩阵

每个节点以卡片形式展示：

| 信息 | 说明 |
|------|------|
| 节点名称 | Kubernetes 节点标识 |
| NPU 状态行 | 每个 NPU 的占用状态可视化 |
| 状态颜色 | 🟢 空闲 / 🔴 占用 / ⚪ 未知 |

### 查看节点详情

点击节点卡片进入节点详情页：

#### 基本信息

| 信息 | 说明 |
|------|------|
| 节点名称 | 节点标识 |
| 状态 | Ready/NotReady |
| 内核版本 | OS 内核版本 |
| 容器运行时 | Docker/containerd |
| kubelet 版本 | Kubelet 版本 |

#### 资源信息

| 资源 | 容量 | 已分配 | 可用 |
|------|------|--------|------|
| CPU | - | - | - |
| 内存 | - | - | - |
| NPU | - | - | - |
| Pod | - | - | - |

#### NPU 详情

对于每个 NPU 显示：

| 信息 | 说明 |
|------|------|
| 设备 ID | NPU 设备标识 |
| 占用状态 | 是否被使用 |
| 占用者 | 使用该 NPU 的用户/作业 |
| 显存利用率 | HBM 使用百分比 |
| 温度 | 设备温度 |
| 功率 | 功耗 |

#### 运行 Pod

显示节点上运行的所有 Pod：

| 信息 | 说明 |
|------|------|
| Pod 名称 | Pod 标识 |
| 命名空间 | 所属命名空间 |
| 状态 | 运行状态 |
| 资源请求 | CPU/内存/NPU 请求 |

## 利用率报告

切换到 **利用率报告** 标签页查看资源利用率分析。

### 时间范围选择

| 选项 | 说明 |
|------|------|
| 1 小时 | 最近 1 小时数据 |
| 6 小时 | 最近 6 小时数据 |
| 24 小时 | 最近 24 小时数据（默认） |
| 7 天 | 最近 7 天数据 |
| 自定义 | 自定义开始和结束时间 |

### 导出报告

点击 **导出报告** 按钮：

1. 选择时间范围
2. 选择数据粒度（原始/按小时/按天）
3. 选择导出格式（CSV）
4. 系统生成报告并自动下载

## 数据刷新

- **手动刷新**：点击右上角 **刷新数据** 按钮
- **自动刷新**：页面每 30 秒自动刷新数据

## 监控指标告警阈值

| 指标 | 正常范围 | 告警阈值 |
|------|----------|----------|
| CPU 使用率 | 0-80% | > 80% |
| 内存使用率 | 0-85% | > 85% |
| NPU 温度 | < 80°C | > 85°C |
| 节点状态 | Ready | NotReady |

## 最佳实践

1. **定期检查节点状态**
   - 关注 NotReady 节点
   - 及时处理异常节点

2. **监控资源使用**
   - 关注资源利用率趋势
   - 识别资源瓶颈

3. **利用碎片资源**
   - 查看各节点 NPU 分布
   - 合理调度任务到空闲节点

4. **定期导出报告**
   - 保存历史利用率数据
   - 用于容量规划和成本分析
