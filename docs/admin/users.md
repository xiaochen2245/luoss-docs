# 用户管理

用户管理页面用于创建、编辑和管理平台用户。

## 功能概述

| 功能 | 说明 |
|------|------|
| **用户列表** | 查看所有用户 |
| **创建用户** | 添加新用户 |
| **编辑用户** | 修改用户信息 |
| **配额管理** | 设置用户配额 |
| **删除用户** | 删除用户账户 |

## 用户列表

| 字段 | 说明 |
|------|------|
| 用户名 | 登录用户名 |
| 邮箱 | 用户邮箱 |
| 角色 | 普通用户/管理员 |
| 用户角色 | 学生用户/普通用户 |
| 状态 | 启用/禁用 |
| 创建时间 | 账户创建时间 |

## 创建用户

### 步骤

1. 点击 **添加用户** 按钮
2. 填写信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用户名 | 是 | 唯一标识符 |
| 密码 | 是 | 初始密码 |
| 邮箱 | 否 | 用于通知 |
| 管理员 | 否 | 是否为管理员 |
| 用户角色 | 否 | 学生用户（默认）或普通用户 |

3. 点击 **创建**

### 用户角色说明

用户角色决定了训练任务的队列分配策略：

| 角色 | 标识 | 队列策略 | 资源保障 |
|------|------|----------|----------|
| 学生用户 | `student` | 个人队列 `user-{username}-compute` | 有保障，独立计算队列 |
| 普通用户 | `normal` | 共享 `normal-queue` | 无保障，与其他普通用户共享资源 |

::: info 默认角色
新建用户默认为"学生用户"角色。管理员可在创建或编辑用户时修改。
:::

### 命名空间创建

创建用户时自动：

1. 创建 Kubernetes 命名空间 `user-{username}`
2. 创建默认 ResourceQuota
3. 初始化用户存储 PVC

### 调试节点 Linux 用户

当启用调试节点功能后，创建用户时还会自动在指定的调试节点上创建同名的 Linux 系统用户，方便用户直接 SSH 登录节点进行调试开发。

::: info 调试节点功能
- 使用 SSH 密钥认证连接调试节点
- 在所有配置的节点上并发创建用户（`useradd -m -s /bin/bash {username}`）
- 创建失败不影响平台用户注册（非阻塞）
- 管理员需预先配置 SSH 密钥 Secret
:::

#### 部署配置

在 `values.yaml` 中配置调试节点：

```yaml
backend:
  debugNode:
    enabled: true
    hosts:
      - "10.1.30.43:22"
      - "10.1.30.44:22"
    sshUser: "root"
    sshKeyPath: "/etc/ktp/ssh/id_rsa"
```

#### 创建 SSH 密钥 Secret

```bash
# 生成密钥对（如果还没有）
ssh-keygen -t ed25519 -f /tmp/ktp-debug-node -N ""

# 将公钥复制到调试节点
ssh-copy-id -i /tmp/ktp-debug-node.pub root@10.1.30.43
ssh-copy-id -i /tmp/ktp-debug-node.pub root@10.1.30.44

# 创建 Kubernetes Secret
kubectl create secret generic k8s-tenant-platform-debug-node-ssh \
  --from-file=id_rsa=/tmp/ktp-debug-node \
  -n k8s-tenant
```

## 编辑用户

### 修改信息

1. 点击 **编辑** 按钮
2. 修改邮箱、角色或用户角色
3. 保存更改

::: info 用户角色变更
修改用户角色后，该用户下次创建训练任务时将使用新的队列策略。已运行的任务不受影响。
:::

### 重置密码

1. 点击 **重置密码**
2. 输入新密码
3. 确认重置

## 配额管理

### 设置配额

1. 点击 **设置配额**
2. 配置资源：

| 资源 | 单位 | 说明 |
|------|------|------|
| CPU | 核 | CPU 配额 |
| 内存 | Gi | 内存配额 |
| NPU | 卡 | NPU 配额 |

3. 保存设置

### 配额模板

| 用户类型 | CPU | 内存 | NPU |
|----------|-----|------|-----|
| 普通用户 | 4 | 8 Gi | 0 |
| 开发人员 | 8 | 16 Gi | 2 |
| 算法工程师 | 16 | 32 Gi | 4 |

## 禁用用户

1. 点击 **禁用** 按钮
2. 确认操作

禁用后：
- 用户无法登录
- 已运行资源继续运行

## 删除用户

::: danger 危险操作
删除用户会：
1. 删除用户数据库记录
2. 删除 Kubernetes 命名空间
3. 删除所有资源和数据
4. **操作不可恢复！**
:::

### 删除步骤

1. 确认用户已停止所有资源
2. 点击 **删除**
3. 输入用户名确认
4. 执行删除
